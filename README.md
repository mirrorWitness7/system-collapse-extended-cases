# System Collapse Taxonomy

A neutral, research-oriented framework for analyzing **structural failure** in organizations, platforms, and socio-technical systems.

This repository focuses on **how systems fail**, not on personalities, blame, or event-driven narratives.  
It is designed for operators, analysts, and researchers who need a shared language for diagnosing collapse patterns across domains.

---

## Purpose

This framework is designed to:

- Map **dimensions** of structural health  
- Define reusable **archetypes** of collapse  
- Catalog recurring **failure modes**  
- Provide a practical **collapse audit template**  
- Demonstrate the framework using **anonymized and fictional case studies**

The emphasis is on *structure over story*.

## Case Design Principles

All cases in this repository are:

- **Pattern-based**, not event-based  
- **Fully anonymized or fictionalized**  
- Written to be **globally applicable**  
- Focused on **structural dynamics**, not intent or morality  

Cases are **additive**: new cases do not redefine the taxonomy â€” they instantiate it.

## Intended Use

This framework is suitable for:

- Organizational and platform audits  
- Governance and risk analysis  
- Post-incident or post-collapse diagnostics  
- Socio-technical system analysis  
- AI and automation lifecycle failure analysis  

It is **not** intended for:
- Real-time incident response  
- Legal adjudication  
- Attribution of personal fault
